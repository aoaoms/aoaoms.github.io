          # 开篇

          vue-element-admin 是一款比较好的开源前端vue开发框架。社区活跃，功能齐全，容易上手。作者刚开始使用其简单模板 vue-element-admin 为基础逐步添加自己的业务功能。此模板已包括了动态路由、登录、element ui和axios 请求的封装； 基本可以开始业务开发。需要其他功能可以直接从 vue-element-admin 复制修改；过程也是更进一步的熟悉使用组件实现业务的好方法。

          :::tips 注意：要下载基础模板包含动态路由控制的话，需要下载其分支：permission-control :::

          昨天看了一遍vue-element-admin 的动态路由控制处理流程，查阅了一下代码实现；这里结合示例说明怎么添加依据其添加自己的业务路由并进行控制。

          有一篇很好的代码透析分享给大家 ：vue-element-admin登录逻辑，以及动态添加路由，显示侧边栏

          # 动态路由处理核心

          可参阅项目根目录下的permission.js 相关代码

          # 添加业务路由

          新增自己的业务路由元信息；文件路径：./src/router/index.js

          /** 这里是动态需要控制的路由元信息
 * asyncRoutes
 * the routes that need to be dynamically loaded based on user roles
 */
export const asyncRoutes = [
  {
    path: '/goods',
    component: Layout,
    redirect: '/goods/goodsCate',
    name: 'Goods',
    meta: {
      title: '商品管理',
      icon: 'nested',
      roles: ['wang']
    },
    children: [
      {
        path: 'goodsCate',
        name: 'GoodsCate',
        meta: { title: '货品类目', roles: ['wang'] },
        component: () => import('@/views/table/index')
      },
      {
        path: 'goodsSpec',
        name: 'GoodsSpec',
        meta: { title: '货品规格', roles: ['wang'] },
        component: () => import('@/views/table/index')
      }
    ]
  }
]
