            # 开篇

            vue-element-admin 是一款比较好的开源前端vue开发框架。社区活跃，功能齐全，容易上手。作者刚开始使用其简单模板 vue-element-admin 为基础逐步添加自己的业务功能。此模板已包括了动态路由、登录、element ui和axios 请求的封装； 基本可以开始业务开发。需要其他功能可以直接从 vue-element-admin 复制修改；过程也是更进一步的熟悉使用组件实现业务的好方法。

            # 关于此篇

            此篇，了解一下框架里面整合的Mockjs 服务。用于模拟接口数据、便于前后端分离开发；提高开发效率。 作者也推荐一个非常好的第三方应用postman ；可以下载桌面应用，注册一下就好了。里面可以完成Mock接口定义，生成API文档，界面非常友好。国内很多仿制他的功能。

            # 简单的理解Mock

            早些年，前后端开发耦合的比较紧；或者说前端还没有划分的这么独立，像现在这么多优秀的技术出现。产品需要一个功能调整或新增功能，具体到新增接口或调整现有功能；从功能设计、界面UI、接口联调等都需要参会沟通确认。并且前端调用接口完全依赖后端的话，十分浪费时间。但是现在，前端有了Mock数据服务的出现，接口测试完全不需要那么依赖后排的兄弟。因为，大家可以按照一份拟定好的接口规则（字段）定义、状态描述等等定义，各自做自己的事情。其实，Mock也不算新事物；现在的Mock服务也只是人们思考解决这种前后端耦合工作带来的麻烦的方法的一种具体工具而已；想一想，如果你想前后分离，也是可以自己维护一份虚拟数据的；只要对标好接口规范就可以。现在出现的很多Mock工具大大简化这方面的工作，提高了工作效率；并且加入了很多更加便利的功能，例如：数据模拟序列化、自动化、缓存、Api文档集成、请求拦截甚至有介入项目管理领域等等。
