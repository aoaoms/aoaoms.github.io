              # 关于微胖viewUpon

              做了几个微信小程序、小游戏小项目，使用了微信云开发（免费的云资源）；后期维护数据发现比较麻烦，需要开发工具登录每个小程序进入云开发管理，于是就有了想做一个一站管理小程序云开发资源（数据）的系统。优点如下：

              1. 多个小程序云资源一站管理，省去使用官方的繁琐操作。
              2. 使用官方标准的API接口处理数据。
              3. 可以自由定制自己的数据展示及处理界面。
              4. 更多可扩展的数据处理及分析功能。

              # 登录\注册\重置密码

              如您看到此文档，则您可能已打开微胖 登录界面；没错，由于使用自有的服务器资源；为了避免接口滥用以及尽可能好的管理资源请求，项目做了注册、登录、重置密码等基础功能。没有账号你可以先注册一个；经过注册信息验证后，你会收到一封含有验证码的邮件，请按照提示进行注册。注册成功后，你就可以登录系统使用微胖啦。(^ ^)。 同样，如果你注册了 但忘了密码时，你可以重置密码；重置密码需要你输入自己的注册邮箱，验证通过后你会收到一封验证码操作重置密码。

              PS: 目前不限定注册用户数量，后续不排除限制注册、限制使用API资源等操作。

              # 环境

              环境对应于你的一个小程序使用的云开发资源环境；微信小程序云开发赋能用户可针对一个小程序创建2个云环境，你可以使用免费的云资源，具体可参阅微信小程序云开发;对于很多小的小程序项目，个人觉得此免费的云资源够用。 在微胖的环境菜单页，你可以操作新增环境、编辑环境、刷新访问凭证token(此凭证用于你访问小程序云api)、移除环境、查看环境集合；并可在右侧集合界面操作集合！ 新增环境需要你添加小程序相关的信息：

              1. 小程序appid
              2. 小程序secret
              3. 其他辅助信息

              在微胖里面这些信息都仅保存在localStorage里面。 新增环境基本信息后，你需要操作【刷新token】；此操作会同步获取最新的小程序云API访问凭证，此凭证是全局的。意思为，如果你的小程序业务逻辑中有其他地方也使用了此token，并进行了缓存；那么这些逻辑即时处理时可能会报错，不是最新的token！更多解释可查阅小程序API凭证。 另外，此token有效期限定2小时。

              为什么要这样设置？ 这些重要数据，微胖不做管理；仅在使用api的时候需要提交请求处理。你需要明确知晓并保管好本地数据。
