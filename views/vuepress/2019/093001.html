              # 目标

              通过浏览器访问 blog.aoaoms.com 个人域名（一级或二级）可以直接跳转到github page页面。

              # 注册github账号

              需要注册github账号, 自行去 github 官网注册就好。

              # 使用vuepress

              使用VuePress-theme-reco 的主题 我这里使用 VuePress-theme-reco 定制的主题，可以访问并操作获取最新版本到本地编辑；过程参见其部署手册，不再赘述。

              # github新建仓库

              新建仓库名时，必须按照特定规范创建；才能达到想要的效果。 例如：你的github用户名是 aoaoms ，那么你创建的仓库名应该写为 aoaoms.github.io 。

              为什么要这样设置？

              因为github page服务针对个人只能有一个可访问首页绑定的域名；并且必须是 master 分支。官方说明如下：

              Source

User pages must be built from the master branch.

              # 配置vuepress

              # package.json

              此文件存在于项目根目录下，打开编辑：

              {
  "name": "blog",
  "version": "1.0.0",
  "repository": "",
  "author": "aoaobaba",
  // 这里的scripts 指定允许npm代理执行的指令代码 即在执行：npm run docs:dev 的时候，
  // 实际上会执行`vuepress dev` docs 
  // 这里定义了2个npm 执行指令，分别是开发模式构建docs、生产构建docs
  "scripts": {
    "docs:dev": "vuepress dev docs",
    "docs:build": "vuepress build docs"
  },
  // 此节点包括项目的依赖包
  "devDependencies": {
    "@vuepress/plugin-medium-zoom": "1.0.3",
    "gh-pages": "^2.1.1",
    "vuepress": "1.0.1",
    "vuepress-plugin-flowchart": "1.4.3",
    "vuepress-theme-reco": "1.0.8-alpha.3"
  },
  "description": ""
}

              # config.js

              此文件存在于 \docs.vuepress 目录下,可以对vuepress项目属性进行配置。这里我更改项目输出的默认目录（个人习惯而已）：

              注意 VuePress-theme-reco 初始设置输出文件在public 一样可以使用，不打井~

              // 这里定义输出构建目录为项目根目录下的dist文件夹
"dest": "./dist",
              1
              2

              重要：网上很多文章说了要设置base属性；这里完全没有必要，设置了反而会有其他问题。如后续在github 中设置page时无法绑定自己的域名到你设置的目录；届时可能仓库的访问地址变成blog.aoaoms.io/blog/ 啦，后续绑定域名就不好处理。就使用vuepress 默认base设置，使用项目根目录。

              # 新增本地仓库文件夹

              重点：这里我在项目根目录下分别创建了2个文件夹 dist 和 blog ;文件夹作用如下：

              • dist

                用途：存放vuepress build构建后的输出文件，因为每次构建都会清空文件夹。

              • blog

                用途：这里存放本地git仓库内容；每次构建完成后，需要从dist 文件夹里面拷贝最新的文件再提交git.

              # 新增部署文件

              可以在项目根目录下创一个sh类型文件，用于shell指令自动部署git。我这里创建了 deploy.sh 文件，如下：

              deploy.sh 内容如下：








               


















              # 确保脚本抛出遇到的错误
set -e

# npm vuepress build doc 执行package.json中定义的scripts 执行指令
npm run docs-build

# 拷贝build文件 到blog 目录 当前目录时blog 以下使用相对路径
cp -r F:/aoaobaba-blog/dist/* F:/aoaobaba-blog/blog

# 进入git仓库文件夹 初始化仓库
cd ./blog

#初始化仓库
git init

#提交所有被修改和新增的文件到缓存区
git add .

# 创建 提交描述
git commit -m 'deploy'

# 如果发布到
git push -f git@github.com:aoaoms/aoaoms.github.io.git master

cd -

              # github设置域名绑定

              在github 打开你的仓库，进入setting设置绑定个人域名：

              绑定域名操作后，会在仓库根目录创建一个CNAME文件，这个是用于绑定解析域名的。每次提交代码到master分支后，可能会删除此文件，导致需要重新操作绑定域名地址。这个时候，可以在CNAME生成好后，复制到本地仓库blog 目录中；进而在后续每次提交的时候，都会再次提交给master分支。
